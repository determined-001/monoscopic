# Function Reference

## `monoscope(apiKey, onEvent, options?)`

Opens an authenticated WebSocket connection to the Monoscope alert stream.

### Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `apiKey` | `string` | Yes | Your API key (`mk_live_...`) from Settings → Developer |
| `onEvent` | `EventCallback` | Yes | Called for every event received from the server |
| `options` | `MonoscopeOptions` | No | Optional configuration |

### Returns

`StopFn` — a function that closes the WebSocket connection and stops reconnect attempts.

```typescript
const stop = monoscope("mk_live_...", (event) => { /* ... */ });

// Later:
stop();
```

### Example

```typescript
import { monoscope } from "@monoscope/sdk";

const stop = monoscope(
  "mk_live_...",
  (event) => {
    switch (event.type) {
      case "ready":
        console.log("Connected");
        break;
      case "alert_triggered":
        console.log(event.data.alert?.name, event.data.trigger?.value);
        break;
      case "error":
        console.error(event.data.message);
        break;
    }
  },
  {
    url: "wss://api.monoscope.xyz",
    reconnectInterval: 3000,
  },
);
```

## `MonoscopeOptions`

```typescript
interface MonoscopeOptions {
  /**
   * WebSocket server URL.
   * Default: "wss://api.monoscope.xyz"
   */
  url?: string;

  /**
   * Milliseconds to wait before reconnecting after a disconnect.
   * Default: 3000
   */
  reconnectInterval?: number;
}
```

---

## `EventCallback`

```typescript
type EventCallback = (event: MonoscopeEvent) => void;
```

Your callback receives a `MonoscopeEvent` on every message. It can be synchronous or `async`.

---

## `StopFn`

```typescript
type StopFn = () => void;
```

Calling `stop()` will:
1. Set an internal `stopped` flag so reconnects no longer trigger
2. Clear any pending reconnect timer
3. Close the WebSocket connection

interface Alert {
